Использование скриптов.

================== Отчет по всем ТБ о количестве подключений АЛФ ==================

usingALFtotal.sql - отчет по всем ТБ о количестве подключений АЛФ на каждую дату
В блоке with проставляем дату начала интервала в блоке select, а также конца и начала интервала в блоке connect. Запускаем в среде разработки запросов.


================== Отчет по каждому ТБ о количестве подключений АЛФ ==================

usingALFbyTB.sql - отчет по каждому ТБ о количестве подключений АЛФ на каждую дату
В блоке with аналогично проставляем интервал дат, а затем через select.. union select.. проставляем список ТБ. Отчет строится по каждому тербанку на весь интервал дат. Запускаем в среде разработки запросов.

================== Отчет по изменению клиентом категории ==================

changingOperationCategory.sql - отчет по изменению клиентом категории карточной операции.
Если отчет формируется по нескольким ТБ, записи в отчете сортируются в начале по ТБ, потом по ВСП, а потом по дате (в порядке возрастания).
Параметры запроса (период отчета, ТБ, ВСП) задаются в блоке where, и, для примера, заполнены значениями, при необходимости их можно скорректировать.

Если необходимо осуществить поиск только по ТБ, то необходимо использовать строчку: and TB in (77, 36). Здесь через запятую перечислются номера ТБ.

Если необходимо осуществить поиск по ТБ и по ВСП, то необходимо использовать строчку: and TB || VSP in (774, 775, 364). Здесь через запятую перечислются номера ТБ склеенные с ВСП. Например, значение "774", означает, что необходимо искать записи, где ТБ - 77, а ВСП - 4.

Если необходимо осуществить поиск по всем ТБ, то необходимо удалить обе эти строки из скрипта.
Скрипт запускается в среде разработки запросов.


================== Отчет по всем ТБ "Популярные цели" ==================

popularTargetsTotal.sql - отчет по всем ТБ, показывающий статистику по всем целям на каждую дату: 
- общее количество кажой цели у всех клиентов 
- среднее количество каждой цели на одного клиента.

Параметры запроса (период отчета) задаются в блоке with: 
- проставляется дата начала интервала в блоке select, а также конца и начала интервала в блоке connect. 

Отчет строится на весь интервал дат. Скрипт запускается в среде разработки запросов.


================== Отчет по каждому ТБ "Популярные цели" ==================

popularTargetsByTB.sql - отчет по каждому ТБ, показывающий статистику по всем целям на каждую дату: 
- общее количество кажой цели у всех клиентов 
- среднее количество каждой цели на одного клиента.

Параметры запроса (период отчета и ТБ) задаются в блоке with: 
- проставляется дата начала интервала в блоке select, а также конца и начала интервала в блоке connect
- через select.. union select.. проставляется список ТБ. 

Отчет строится по каждому тербанку на весь интервал дат. Скрипт запускается в среде разработки запросов.


================== Отчет по использованию параметров фильтра в расходах АЛФ по всем ТБ ==================

outcomeFilterParams.sql - отчет по использованию параметров фильтра в расходах АЛФ по всем ТБ
В блоке with задаются параметры интервала дат. В блоке select проставляется дата начала интервала, а в блоке connect задаются конец и начало интервала.
Вывод происходит по каждой дате интервала. На каждую дату выводится количество использований фильтра по каждому типу. Даты отсортированы в порядке возрастания.
Все параметры запроса заданы для примера и могут быть скорректированы.


================== Отчет по использованию параметров фильтра в расходах АЛФ по заданным ТБ ==================

outcomeFilterParamsByTb.sql - отчет по использованию параметров фильтра в расходах АЛФ по заданным ТБ
Аналогично отчету outcomeFilterParams задаётся интервал дат. Перечисление тербанков осуществляется через select 'номер тербанка' from dual union select 'номер тербанка 2' from dual 
Отчет сортируется вначале по тербанкам, затем по датам. Вывод происходит по каждой дате интервала. Все параметры запроса заданы для примера и могут быть скорректированы.


================== Отчет по автоматической перекатегоризации ==================

recategorizationStatistic.sql - отчет по автоматической перекатегоризации. 
Входным параметром отчета является период, за который должны отобразиться данные.
