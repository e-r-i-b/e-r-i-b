			ИНСТРУКЦИЯ
	Создание(обновление) заглушенной базы 

Для создания базы необходимо выполнить скрипт ips_create_obj.sql под пользователем, имеющим права администратора. Скрипт создает все необходимые структуры (схема, таблица, хранимые процедуры). 
Если этот скрипт уже выполнялся, то нужно выполнить скрипт для обновления ips_update_obj.sql

ВНИМАНИЕ!!! Перед выполнение необходимо указать входные параметры.
Скрипт позволяет указать:

- "c_User" - схема, которая будет создана для заглушенной базы.
------------------------------------------------
	Заполнение созданной базы тестовыми данными
	
Необходимо выполнить скрипт ips_data_load.sql. Скрипт заполняет таблицу транзакций по картам тестовыми данными. 

ВНИМАНИЕ!!! Перед выполнение необходимо указать входные параметры.
Данный скрипт позволяет изменять:

- "c_User" - схема для загрузки данных.
- "c_TransactionCount" - общее кол-во транзакций, т.е. сколько всего записей будет в таблице. 
  Чтобы изменить кол-во транзакций которые будут сгенерированны скриптом, необходимо поменять значение "c_TransactionCount".
- "c_TransactionDateBegin" - дата начала генерации транзакций. 
  Данный параметр указывает с какой даты начнут формироваться транзакции.
  
  Необходимо указать либо "c_TransactionCount" либо "c_TransactionDateBegin".
  Если указана "c_TransactionDateBegin", то "c_TransactionCount" высчитывается автоматически.
  Если указано "c_TransactionCount", то автоматически высчитывается "c_TransactionDateBegin".
  Если указаны оба параметра, то во внимание принимается "c_TransactionDateBegin" и происходит корректировка "c_TransactionCount".
  
- "c_CardTransacToDayCount" - максимальное кол-во транзакций в день для одной карты.

- "c_DocNum" - номер документа.

- "c_CardList" - список карт через ",". 
  Данный список содержит список номеров карт, по которым будут генериться транзакции.
  Кол-во карт, которые можно указать в списке не ограничено. Номера кат необходимо отделять запятой.

- "c_ISOList" - список ISO кодов валюты счета через ",". 
  Требования аналогичны списку карт. Кол-во кодов должно соответствовать кол-ву карт.

- "c_CardAccauntNumList" - список номеров счетов карт через ",". 
  Требования аналогичны списку карт. Кол-во должно соответствовать кол-ву карт.

- "c_ISOTransList" - список ISO кодов валюты транзакции четез ",". 
  Требования аналогичны списку карт. Кол-во может не соответствовать с кол-вом карт.

- "c_TransDescription" - список описаний операций с местом ее совершения. 
  Требования аналогичны списку карт. Кол-во может не соответствовать с кол-вом карт.

- "c_MCCCodeList" - список MCC-кодов. 
  Требования аналогичны списку карт. Кол-во может не соответствовать с кол-вом карт.
  
Все параметры находятся в начале скрипта и, для примера, заполнены значениями, при необходимости их можно скорректировать.
