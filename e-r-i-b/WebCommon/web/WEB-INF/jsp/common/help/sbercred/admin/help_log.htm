Пункт меню Сервис позволяет Вам выполнять различные сервисные операции, например, просматривать журналы, в которых регистрируются действия сотрудников и клиентов банка, менять свой пароль для входа в систему, загружать новые формы платежей и т.д.

Журнал действий пользователей

Фильтр операций

Пункт меню Журнал действий пользователей - протокол действий пользователей, в который занесены все операции, выполненные пользователями системы.

Данный пункт меню используется для контроля за действиями клиентов и сотрудников банка в системе. Он предназначен для администраторов и сотрудников, которые занимаются аудитом.

В журнале действий пользователей по каждой совершенной операции можно просмотреть следующую информацию:

  • "Номер записи" - порядковый номер записи.

  • "Дата и время" - дата и время совершения операции.

  • "Приложение" - отображается приложение клиента банка (для клиента) или приложение арм-сотрудника (для сотрудника).

  • "Идентификатор пользователя" - идентификационный номер пользователя.

  • "ФИО" - фамилия, имя, отчество пользователя системы.

  • "Подразделение" - подразделение, за которым закреплен определенный пользователь.

  • "Наименование операции" - название операции, выполненной пользователем.

  • "Параметры операции" - параметры операции, выполненной клиентом или сотрудником банка.

  • "IP-адрес" - IP-адрес пользователя системы.

  • "Идентификатор сессии" - уникальный идентификатор сессии пользователя.

  • "Тип сообщения " - в системе сообщения делятся на несколько типов: "информационное" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "безопасность" (сообщение подсистемы безопасности о том, что пользователь совершил операцию, доступ к которой ему запрещен), "ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

Фильтр операций

Для просмотра списка всех интересующих Вас операций, выполненных пользователем системы, щелкните по кнопке Показать фильтр и заполните поля фильтра:

  • "ФИО" - поиск по имени пользователя;

  • "Период" - поиск по дате и времени, за которое были совершены операции в системе;

  • "Приложение" - поиск по приложению, в котором пользователем была совершена операция;

  • "Идентификатор пользователя" - поиск по идентификационному номеру пользователя в системе;

  • "Подразделение" - поиск по номеру подразделения, за которым закреплен пользователь системы;

  • "IP-адрес" - поиск по IP-адресу пользователя;

  • "Наименование операции" - поиск по названию операции;

  • "Тип сообщения " - поиск по типу сообщения: "информационное" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "безопасность" (сообщение подсистемы безопасности о том, что пользователь совершил операцию, доступ к которой ему запрещен), "ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

  • "Максимальное количество записей " - настройка отображения, определяющая, сколько записей будет выведено на страницу. Если все требуемые записи не помещаются на одной странице, то будет произведена разбивка по страницам с указанным количеством записей.

В фильтре автоматически задается период выполненных действий за прошедшие сутки, но сотрудник банка может задавать любой интересующий его период, за который были выполнены действия пользователя системы.

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв фамилии, имени или отчества пользователя.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать всех пользователей с фамилией на букву "А" из одного подразделения банка.

После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас операций пользователей.

Если Вам необходимо выгрузить определенные записи журнала в файл, щелкните по кнопке Выгрузить, откроется окно, в котором нужно указать операцию Сохранить на диск и затем выбрать или ввести имя файла для сохранения.

Журнал системных действий

Фильтр операций

Журнал системных действий используется для фиксации ошибок и сбоев, произошедших при работе пользователей в системе, запросов, пришедших от пользователей, задач, запущенных по расписанию, для отслеживания попыток взлома системы, а также содержит отладочную информацию. Он предназначен для системных администраторов и сотрудников банка, которые занимаются аудитом.

В журнале системных действий можно просмотреть следующую информацию:

  • "Номер записи" - порядковый номер записи,

  • "Дата и время" - дата и время совершения операции.

  • "Тип сообщения" - тип сообщения: "информация" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "диагностика" (сообщение, предупреждающее о возможности сбоя, ошибки в системе), "ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

  • "Приложение" - приложение, в котором совершена операция: арм-сотрудника, клиента, обработка оповещений и др.

  • "Идентификатор пользователя" - идентификационный номер пользователя системы;

  • "ФИО" - фамилия, имя, отчество пользователя системы;

  • "IP-адрес" - IP-адрес пользователя системы;

  • "Идентификатор сессии" - уникальный идентификатор сессии пользователя;

  • "Сообщение" - текст сообщения системы.

Фильтр/поиск операций

Для просмотра списка всех интересующих Вас операций системы щелкните по кнопке Показать фильтр и заполните поля фильтра:

  • "ФИО" - поиск по имени пользователя системы,

  • "Период" - поиск по дате и времени, за которое были выполнены системные действия;

  • "Приложение" - поиск по приложению, в котором были выполнены системные действия;

  • "Идентификатор пользователя" - поиск по идентификационному номеру пользователя в системе;

  • "Идентификатор ошибки" - поиск по идентификатору ошибки, случившейся в результате работы пользователей в системе. Данный идентификатор отображается пользователю на экране при возникновении ошибки.

  • "IP-адрес" - поиск по IP-адресу пользователя;

  • "Слово в тексте" - поиск по слову в тексте сообщения системы;

  • "Тип сообщения" - поиск по типу сообщения системы: "информация" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "диагностика" (сообщение, предупреждающее о возможности сбоя, ошибки в системе), "ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

  • "Максимальное количество записей " - настройка отображения, определяющая, сколько записей будет выведено на страницу. Если все требуемые записи не помещаются на одной странице, то будет произведена разбивка по страницам с указанным количеством записей.

В фильтре автоматически задается период выполненных действий за прошедшие сутки, но Вы можете задать любой интересующий Вас период, за который были выполнены системные действия.

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв фамилии, имени или отчества пользователя.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать сообщения по приложению, в котором совершена операция, и по слову в тексте сообщения.

После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас операций, совершенных системой.

Если Вам необходимо выгрузить определенные записи журнала в файл, щелкните по кнопке Выгрузить, откроется окно, в котором нужно указать операцию Сохранить на диск и затем выбрать или ввести имя файла для сохранения.

Журнал сообщений

Фильтр операций

В пункте меню Журнал сообщений (журнал взаимодействий с внешними системами) отображаются сообщения между системой ИКФЛ и RS-Retail.

В журнале сообщений по каждому сообщению, полученному от RS-Retail или отправленному внешней системе, можно просмотреть следующую информацию:

  • "Номер записи" - номер сообщения.

  • "Дата и время" - дата и время сообщения.

  • "Приложение" - приложение, которое отправило сообщение во внешнюю систему.

  • "Система" - система, которой адресовано сообщение (исходящее), или система, от которой получено сообщение (входящее).

  • "Тип запроса" - вид XML-сообщения, которое отправляется из системы ИКФЛ в RS-Retail.

  • "Тип ответа" - вид XML-сообщения, которое отправил RS-Retail на запрос.

  • "Сообщения" - содержание сообщения системы. Для того чтобы просмотреть содержание сообщения, щелкните по кнопке <...>, откроется страница просмотра сообщения, на которой система покажет Вам запрос ИКФЛ во внешнюю систему и ответ на запрос.

Фильтр/поиск сообщений

Для просмотра списка всех интересующих Вас сообщений щелкните по кнопке Показать фильтр и заполните поля фильтра:

  • "Тип запроса" - поиск по виду (наименованию) XML-сообщения, которое было отправлено из системы ИКФЛ в RS-Retail. Тип отражает, что за сообщение было отправлено или получено шлюзом системы: запрос на получение остатка денежных средств на счете, запрос и получение остатка денежных средств на пластиковой карте, запрос и получение комиссии по платежной операции и т.д.

  • "Тип ответа" - поиск по виду (наименованию) XML-сообщения, которое было получено системой ИКФЛ от RS-Retail.

  • "Система" - система, которой адресовано сообщение (исходящее), или система, от которой получено сообщение (входящее).

  • "Период" - поиск по дате и времени, за которое были отправлены и получены сообщения системой;

  • "Слово в запросе" - поиск по слову в запросе,

  • "Слово в ответе" - поиск по слову в ответе,

  • "Приложение" - приложение, которое отправило сообщение во внешнюю систему.

  • "Максимальное количество записей " - настройка отображения, определяющая, сколько записей будет выведено на страницу. Если все требуемые записи не помещаются на одной странице, то будет произведена разбивка по страницам с указанным количеством записей.

В фильтре автоматически задается период выполненных действий за прошедшие сутки, но Вы можете задавать любой интересующий Вас период, за который были получены и отправлены сообщения.

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв фамилии, имени или отчества пользователя.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать сообщения по приложению, в котором совершена операция, и по слову в ответе.

После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас сообщений.

Если Вам необходимо выгрузить определенные записи журнала в файл, щелкните по кнопке Выгрузить, откроется окно, в котором нужно указать операцию Сохранить на диск и затем выбрать или ввести имя файла для сохранения.

Сменить пароль

Чтобы изменить пароль для входа в систему, в поле "введите новый пароль" впишите Ваш новый пароль, потом подтвердите новый пароль, заполнив поле "повторите новый пароль" той же комбинацией символов. Затем щелкните по кнопке Изменить, откроется окно, в котором будет написано "пароль на вход в систему изменен". В результате при следующем входе в систему Вы должны будете указать в поле "пароль" Ваш новый пароль.

Время приема документов

Пункт меню Время приема документов предназначен для настройки режима приема платежных документов от клиентов банка. В данном пункте меню задаются границы времени приема документов в течение дня, которые определяют, каким днем будут приниматься платежные документы. Например, если для рублевого платежа Вы укажете время приема документов 17.00, то документы, полученные позже 17.00, будут обрабатываться системой следующим операционным днем.

По умолчанию режим приема документов задан. Вы можете посмотреть и при необходимости изменить настройки. Для изменения настройки выберите документ, установив напротив него галочку, и щелкните по кнопке Редактирование. Откроется страница для редактирования времени, на которой введите в поле новое значение, время должно быть в формате ЧЧ:ММ, и щелкните по кнопке Изменить. В результате время приема выбранного Вами документа будет изменено в системе. Чтобы вернуться на страницу со списком времени приема документов, щелкните по кнопке Список.

Формы платежей

Добавление новой формы платежа

В пункте меню Формы платежей содержится список форм платежей, из которого Вы можете добавить клиенту различные виды платежей.

Добавление новой формы платежа

Создать новый платеж можно на основе одного из уже существующих платежей.

Надо выбрать среди имеющихся платежей наиболее близкий по составу полей. Затем следует выгрузить файлы выбранного платежа на диск.

Чтобы выгрузить новый платеж, надо:

  • выбрать платеж в списке, установив галочку напротив интересующего Вас платежа;

  • щелкнуть по кнопке Выгрузить. Откроется страница, на которой находятся ссылки на файлы, которые нужно выгрузить. Для того чтобы выгрузить файлы, щелкните по каждой ссылке и нажмите кнопку Сохранить.

  • переименовать выгруженные файлы;

  • внести необходимые изменения;

  • затем загрузить новый платеж.

Чтобы загрузить новый платеж, нужно:

  • выбрать раздел главного меню Сервис, пункт Формы платежей,

  • щелкнуть по кнопке Загрузить,

  • указать файлы на диске.

Для того чтобы платеж появился в клиентском приложении, добавьте его на страницу списка платежей и на страницу списка шаблонов.

Загрузка справочников

С помощью пункта меню Загрузка справочников Вы можете загрузить в систему ИКФЛ справочники из системы RS-Retail, которые отображены на экране.

В систему можно загрузить следующие справочники:

  • Справочник валют, в котором содержатся коды валют.

  • Справочник иностранных банков, содержащий реквизиты иностранных банков.

  • Справочник банков, в котором содержится информация о реквизитах банков.

  • Справочник стран, содержащий коды стран.

  • Справочник КБК, в котором содержатся коды бюджетной классификации.

  • Справочник офисов, в котором содержится информация об офисах банка: местоположение, контактные телефоны, график работы.

  • Справочник кредитных офисов, содержащий информацию о дополнительных офисах, в которых предоставляются кредиты клиентам банка.

Чтобы загрузить справочники, выделите галочкой нужный справочник или все справочники и щелкните по кнопке Загрузить. Когда справочник загрузится, система покажет Вам сообщение "Справочники загружены".

Справочник ошибок

Фильтр редактирования сообщения об ошибке

Пункт меню Справочник ошибок используется для просмотра, создания и изменения сообщений системы об ошибках пользователей. Он предназначен для обработки сообщений, полученных от внешних систем Рапида, RS-Retail об ошибках клиентов. В данном пункте задается соответствие между текстом сообщения из внешней системы и интерпретацией его в системе ИКФЛ. Например, если при ошибке из RS-Retail получено сообщение "(662)|(411)", то клиенту придет сообщение "Недостаточно средств на счете списания".

Для того чтобы создать новое сообщение об ошибке, щелкните по кнопке Создать, откроется форма редактирования сообщения. В данной форме в поле "тип ошибки" по умолчанию задается значение "клиентская", в поле "система" выберите систему, из которой получено сообщение об ошибке, в поле "регулярное выражение" укажите регулярное выражение ошибки, присланной из внешней системы, в поле "сообщение в ИКФЛ" введите выражение, которое отправляется клиенту банка при ошибке в системе. Затем нажмите кнопку Сохранить, и новое сообщение об ошибке появится в списке.

Для изменения сообщения установите галочку напротив интересующего сообщения и щелкните по кнопке Изменить или кликните на названии ошибки, система покажет Вам форму редактирования сообщения. В данной форме измените значение в интересующем Вас поле и щелкните по кнопке Сохранить. Если Вы не хотите менять сообщение об ошибке, то щелкните по кнопке Отменить.

Если Вы хотите удалить какое-либо сообщение об ошибке клиента, установите галочку напротив интересующего Вас сообщения и щелкните по кнопке Удалить. Система попросит Вас подтвердить операцию, Вы подтверждаете, и выбранная ошибка будет удалена из списка.

Фильтр редактирования сообщения об ошибке

Для просмотра списка всех интересующих Вас сообщений об ошибке пользователя системы щелкните по кнопке Показать фильтр и заполните поля фильтра:

  • "Регулярное выражение" - поиск по регулярному выражению ошибки;

  • "Тип ошибки" - поиск по типу ошибки пользователя;

  • "Система" - поиск по названию системы, в которой допущена ошибка.

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых регулярного выражения ошибки.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать сообщения по системе, в которой совершена ошибка, и по типу ошибки.

После того как искомые параметры заданы, щелкните по кнопке Сохранить, и система покажет список интересующих Вас сообщений.

Шаблоны документов

Пункт меню Шаблоны документов предназначен для управления шаблонами документов.

Шаблон представляет собой обычный документ Microsoft Word, в котором есть статический текст и специальные теги. Статический текст остается неизменным, а теги при просмотре и печати документа заменяются информацией, которая берется из базы данных системы.

Для того чтобы создать шаблон документа, щелкните по кнопке Добавить документ, откроется страница создания шаблона документа. На данной странице в поле "наименование" введите название шаблона, в поле "описание" укажите сведения о шаблоне, в поле "файл" щелкните по кнопке Обзор, выберите файл с шаблоном, нажмите кнопку Сохранить документ, и шаблон документа появится в списке шаблонов документов. Если Вы не хотите создавать шаблон документа, щелкните по кнопке Список документов, и система вернет Вас в список шаблонов документов.

Если Вы хотите удалить шаблон документа, установите галочку напротив интересующего Вас шаблона, щелкните по кнопке Удалить документ, и шаблон будет удален из списка.

Шаблоны документов можно группировать в пакеты шаблонов (группы шаблонов документов). См. раздел справки "Пакеты шаблонов".

Пакеты шаблонов

Пункт меню Пакеты шаблонов предназначен для хранения списка пакетов шаблонов договоров. Пакет шаблонов формируется из шаблонов документов. Пакет договоров определяет перечень шаблонов документов (договоров), которые печатаются и подписываются клиентом при подключении к системе. Сотрудник банка назначает клиенту пакет договоров.

Для того чтобы создать пакет шаблонов договоров, щелкните по кнопке Добавить пакет, откроется страница создания пакета документов. На данной странице в поле "наименование" введите название пакета, в поле "описание" укажите сведения о пакете, прикрепите один или несколько шаблонов документов, установив напротив них галочки, щелкните по кнопке Сохранить пакет, и пакет шаблонов появится в списке. Если Вы не хотите создавать пакет шаблонов, щелкните по кнопке Список пакетов, и система вернет Вас в список пакетов шаблонов договоров.

Если Вы хотите удалить пакет шаблонов, установите галочку напротив интересующего Вас пакета шаблонов договоров, щелкните по кнопке Удалить пакет, и пакет шаблонов будет удален из списка.

Документы банка

Пункт меню Документы банка предназначен для хранения списка документов банка, которые клиент может скачать, кликнув на соответствующей ссылке в пункте меню Заявки или Платежи.

В данный пункт меню загружаются типовые договоры вкладов и счетов, для того чтобы клиент банка смог ознакомиться с их текстом при подаче заявки на открытие счета/вклада. Файлы могут быть любого формата: Word, Excel, PDF и пр. Клиент банка может скачать файл и открыть его предназначенной для этого программой.

Для того чтобы создать документ банка, щелкните по кнопке Добавить документ, откроется страница создания шаблона документа. На данной странице в поле "наименование" введите название документа, в поле "описание" укажите сведения о документе, в поле "файл" щелкните по кнопке Обзор, выберите файл с документом, нажмите кнопку Сохранить документ, и документ банка появится в списке. Заданные поля будут видны клиентам банка при скачивании шаблона документа. Если Вы хотите просмотреть документ, нажмите кнопку Просмотр. Если Вы не хотите создавать документ, щелкните по кнопке Список документов, и система вернет Вас в список документов банка.

Если Вы хотите удалить документ из списка, установите галочку напротив интересующего Вас документа, щелкните по кнопке Удалить документ, и он будет удален из списка.
