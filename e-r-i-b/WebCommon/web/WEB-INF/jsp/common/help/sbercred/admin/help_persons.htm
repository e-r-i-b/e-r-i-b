Список клиентов

Фильтр/поиск клиентов

Пункт меню Клиенты предназначен для управления пользователями системы. Он позволяет регистрировать новых клиентов (импортировать данные клиентов из установленной в банке системы розничного обслуживания), удалять, блокировать и разблокировать клиентов, управлять доступом клиентов к счетам и картам, назначать клиентам схемы доступа, определяющие доступные им операции, управлять картами ключей клиентов.

В данном пункте меню автоматически открывается список всех клиентов, закрепленных за определенным подразделением банка.

Фильтр/поиск клиентов

Для облегчения поиска требуемого клиента в системе предусмотрен фильтр. Чтобы воспользоваться фильтром, щелкните по кнопке Показать фильтр и задайте в появившихся в верхней части списка полях критерии поиска:

  • "ФИО клиента" - поиск по имени клиента;

  • "Вид документа" - поиск по виду документа, удостоверяющего личность клиента;

  • "Серия" - поиск по серии паспорта клиента банка;

  • "Номер" - поиск по номеру паспорта клиента банка;

  • "Номер пин-конверта" - поиск по номеру пин-конверта клиента банка;

  • "Номер договора" - поиск по номеру договора клиента банка;

  • "Статус" - поиск по статусу клиента банка в системе ("Активный", "Заблокирован", "Подключение").

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв фамилии, имени или отчества клиента.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать всех активных клиентов с фамилией на букву "А" с паспортом.

После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас клиентов банка.

В пункте меню Группы Вы можете просмотреть список групп клиентов банка, создать, отредактировать или удалить группу. Принадлежность клиентов банка к определенной группе позволит Вам отправлять письма клиентам, объединенным в группы. По каждой группе система показывает название группы и ее описание.

После того как параметры заданы, нажмите кнопку Применить, и система покажет список интересующих Вас клиентов, входящих в состав группы.

Для создания новой группы клиентов щелкните по кнопке Добавить. Откроется окно редактирования группы, в котором Вы вводите в поле "наименование" название группы и при необходимости указываете в поле "описание" комментарий к группе. Если Вы не хотите редактировать сведения о группе, щелкните по кнопке Список , и система вернет Вас в список групп клиентов банка.

  • "Серия" - поиск по серии документа, удостоверяющего личность клиента;

  • "Номер" - поиск по номеру документа, удостоверяющего личность клиента;

  • "Номер пин-конверта" - поиск по номеру пин-конверта, присвоенного клиенту;

  • "Номер договора" - поиск по номеру договора клиента с банком;

  • "Статус" - поиск по статусу клиента в системе: "Активный", "Заблокирован", "Подключение".

    Чтобы включить клиента банка в группу, щелкните по кнопке Добавить , откроется страница со списком клиентов, из которого выберите интересующего Вас клиента, установите напротив него галочку и щелкните по кнопке Выбрать . В результате система включит данного клиента банка в состав группы.

    Удаление группы

    Если Вы хотите удалить группу клиентов, поставьте флажок напротив той группы, которую будете удалять, и щелкните по кнопке Удалить. Система попросит Вас подтвердить удаление, и после подтверждения группа будет удалена из списка групп клиентов банка.

Подключение клиента

Фильтр/поиск клиентов

Чтобы зарегистрировать нового клиента, Вам нужно выполнить следующие действия:

  1. В главном меню системы щелкните по пункту Клиенты. На экран будет выведен список уже зарегистрированных в системе клиентов.

  2. Щелкните по расположенной над списком пользователей кнопке Добавить. На экран будет выведен список клиентов, зарегистрированных в системе автоматизации розничных банковских услуг. В данном списке Вы можете осуществить поиск клиентов банка, сведения о которых хранятся в системе Ритейл.

    3. Фильтр/поиск клиентов

    Для облегчения поиска требуемого клиента в системе предусмотрен фильтр. Чтобы воспользоваться фильтром, щелкните по кнопке Показать фильтр и задайте в появившихся в верхней части списка полях критерии поиска:

    • "ID" - поиск клиента по идентификатору в САРБУ (системе автоматизации розничных банковских услуг);

    • "Фамилия, имя и отчество" - поиск по имени клиента банка.

    Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв фамилии, имени или отчества клиента.

    Также Вы можете фильтровать список по любой комбинации полей.

    После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас клиентов банка.

Общие сведения

В данной вкладке анкеты клиента банка вводятся сведения о клиенте. Основную информацию о клиенте Вы импортировали из САРБУ. Остальные сведения Вы вводите самостоятельно. Обязательны для заполнения все поля, отмеченные звездочкой: кем выдан удостоверяющий личность клиента документ, гражданство, секретное слово клиента, номер пин-конверта, дата начала обслуживания клиента.

Примечание: если Вы не имеете пин-конверт для данного клиента, его необходимо создать в системе.

Чтобы создать и привязать клиенту банка пин-конверт, выберите пункт главного меню Пин-конверты, введите количество пин-конвертов в запросе и подразделение банка, за которым закреплен клиент, выгрузите запрос на генерацию ПИН-конверта, щелкнув по кнопке Выгрузить.

Откроется стандартное окно Windows Сохранить . Сохраните файл на сменный носитель(диск, "флешку"), отнесите его на сервер, где установлена утилита печати пин-конвертов, скопируйте файл в каталог XML_in, запустите утилиту печати и распечатайте пин-конверты.

Чтобы загрузить ответ из утилиты печати, Вы должны на сервере, где установлена утилита печати, из каталога XML_out скопировать файл ответа на сменный носитель, отнести его на компьютер, где запущена система ИКФЛ, и указать путь до файла с ответом.

Вы имеете возможность сохранить частично заполненную анкету. Для этого щелкните по кнопке Сохранить .

Чтобы удалить клиента из системы, необходимо расторгнуть договор с данным клиентом на предоставление услуг Интернет-банкинга. Для этого Вам нужно в анкете клиента банка в поле "дата расторжения договора" ввести дату расторжения, щелкнуть по кнопке Сохранить и затем щелкнуть по кнопке Удалить. В результате клиент будет удален из списка клиентов банка и появится в списке архива клиентов банка.

Если Вы хотите выйти из анкеты клиента, щелкните по кнопке Список.

Счета и карты

Для каждого пользователя необходимо создать список счетов и пластиковых карт, к которым он будет иметь доступ. Обратите внимание, что при импорте анкеты клиента банка из системы розничного обслуживания при загрузке автоматически загружаются все имеющиеся там счета и пластиковые карты данного клиента. Причем клиент будет иметь доступ ко всем этим счетам и картам, так что может потребоваться удалить лишние счета и/или карты из списка.

Чтобы добавить или удалить счета клиента из системы выполните следующие действия:

  • В анкете клиента выберите пункт Счета и карты, на экран будет выведен список счетов и карт, к которым клиент имеет доступ.

  • Чтобы добавить в список счета (карты), щелкните по расположенной над списком кнопке Добавить счет (если Вы хотите добавить сведения о пластиковых картах, то надо щелкнуть по кнопке Добавить карту).

Права доступа

Вы должны для клиента задать доступные способы аутентификации: "упрощенный" с помощью логина/пароля и "защищенный" с аутентификацией по сертификату ключа ЭЦП и для каждого из них указать:

  1. Схему прав доступа, т.е. перечень операций, доступных клиенту при входе в систему данным способом. Например, при аутентификации по сертификату пользователь получает права на выполнение всех операций, включая платежные, а при входе в систему по логину и паролю он получает только информационный доступ.

  2. Способ подтверждения входа: при упрощенном доступе Вы можете установить способ "без подтверждения", т.е. клиент банка входит в систему только с помощью логина/пароля, или с подтверждением одноразовым паролем с карты ключей; при защищенном доступе вход клиента в систему осуществляется с помощью сертификата (ЭЦП).

  3. Способ подтверждения операций в системе.

Чтобы назначить клиенту схему прав доступа, щелкните в анкете клиента по вкладке Права доступа и в появившейся форме выберите одну из имеющихся схем доступа или задайте индивидуальную схему, установив флажки у тех операций, которые будет осуществлять данный клиент и щелкните по кнопке Сохранить .

Список представителей

Во вкладке анкеты Представители Вы можете просмотреть, добавить или изменить по желанию клиента представителей, которые имеют доступ к выбранным счетам, картам и операциям клиента в системе.

Регистрация представителя клиента

Подключить нового представителя клиенту банка Вы можете во вкладке анкеты клиента Представители, щелкнув по кнопке Добавить.

Откроется анкета создания нового представителя, в которой Вы заполняете ФИО, дату рождения, серию и номер документа, удостоверяющего личность представителя клиента, дату выдачи, кем выдан документ, гражданство, номер пин-конверта и дату начала обслуживания. Обязательны для заполнения все поля, отмеченные звездочкой.

Примечание: если Вы не имеете пин-конверт для представителя клиента банка, Вы должны создать пин-конверт в системе и привязать его представителю. Для этого выберите пункт главного меню Пин-конверты, введите количество пин-конвертов в запросе и подразделение банка, за которым закреплен клиент, выгрузите запрос на генерацию ПИН-конверта, щелкнув по кнопке Выгрузить.

Откроется стандартное окно Windows Сохранить. Сохраните файл на сменный носитель(диск, "флешку"), отнесите его на сервер, где установлена утилита печати пин-конвертов, скопируйте файл в каталог XML_in, запустите утилиту печати и распечатайте пин-конверты.

Чтобы загрузить ответ из утилиты печати, Вы должны на сервере, где установлена утилита печати, скопировать из каталога XML_out файл ответа на сменный носитель, отнести его на компьютер, где запущена система ИКФЛ, и указать путь до файла с ответом.

Вам необходимо выбрать доступные представителю счета и карты клиента банка, для этого поставьте флажок напротив пункта Перечень доступных счетов или Перечень доступных карт. Вы должны также указать список доступных данному представителю операций в системе:

  • если нужно разрешить все операции из списка, поставьте флажок напротив пункта Перечень разрешенных операций и щелкните по кнопке Сохранить .

  • если нужно разрешить только определенные операции, поставьте флажки напротив выбранных операций и щелкните по кнопке Сохранить .

Чтобы активировать представителя клиента банка, в анкете представителя щелкните по кнопке Подключить клиента . Если все поля анкеты со звездочкой заполнены правильно, то представителю клиента система присвоит статус "Активный".

Группы

Вы можете включить клиента в состав группы, для этого установите флажок напротив той группы, в которую хотите включить клиента, и щелкните по кнопке Сохранить . Включение клиента в группу дает возможность отправлять письма группе клиентов.

Карты ключей

С помощью карт ключей клиент банка может подтвердить доступ в систему и совершение операций.

Чтобы привязать и активировать карту ключей клиенту банка, щелкните по кнопке Добавить, введите номер карты, которую нужно добавить клиенту банка, и снова нажмите кнопку Добавить. Карта ключей будет добавлена клиенту. Если у клиента нет в данный период активной карты, то карте система сразу присвоит статус "Активна". Если добавленная карта ключей неактивна, чтобы ее активировать щелкните по кнопке Активировать. У клиента может быть активна только одна карта ключей, если Вы хотите активировать другую карту ключей, то предыдущая карта автоматически заблокируется.

Если у клиента нет карты ключей, ее надо создать. Создать карту ключей можно типографским способом и через утилиту печати.

  1. Создание карт ключей типографским способом (пароли карты ключей наносятся под защитный слой скретч-карты).

    Чтобы создать карту ключей, откройте пункт меню Карты ключей - Генерация карт ключей, в форме создания карт ключей в поле "количество карт в запросе" укажите количество карт, в поле "количество ключей в карте" введите количество ключей на карте, щелкните по кнопке Создать. Далее сохраните файл на диске, который будет содержать всю необходимую информацию для печати скретч-карты.

  2. Создание карт ключей через утилиту печати:

    Для создания карты ключей данным способом откройте пункт Карты ключей - Заявка на печать карт ключей, в форме для создания запроса на печать в поле "количество карт в запросе" введите количество карт, в поле "количество ключей в карте" укажите число ключей, рядом с полем "подразделение" нажмите кнопку (...), выберите из списка подразделение банка, для которого изготавливаются карты ключей по данному запросу, нажмите кнопку Выбрать. После заполнения полей формы щелкните по кнопке Выгрузить. Откроется стандартное окно Windows Сохранить. Вы должны сохранить файл на сменном носителе (флешке, диске), отнести на сервер, где установлена утилита печати пин-конвертов, скопировать файл в каталог XML_in, запустить утилиту и распечатать карту ключей.

    Чтобы загрузить обработанную заявку в систему, Вы должны на сервере скопировать файл с ответом из утилиты печати из каталога XML_out на сменный носитель, с которого копируете файл ответа на компьютер, где установлена система ИКФЛ, открываете пункт Карты ключей - Загрузка обработанной заявки, нажимаете кнопку Обзор, выбираете путь до файла с ответом и щелкаете по кнопке Выполнить импорт.

    Сразу после создания запроса новая карта ключей появляется в общем списке карт ключей. Вы должны указать номер этой карты в анкете клиента. Для этого в пункте главного меню Клиенты щелкните на имени клиента, откроется анкета клиента, выберите вкладку Карты ключей, нажмите кнопку Добавить, введите номер созданной Вами карты ключей и снова щелкните по кнопке Добавить. Карта ключей будет добавлена клиенту. Данной карте система сразу присвоит статус "Активна".

С помощью карт ключей клиент банка может подтвердить дополнительно доступ в систему и совершение операций.

Сертификаты

В данной вкладке анкеты клиента открывается список всех сертификатов клиента. По каждому сертификату Вы можете просмотреть:

  • "Идентификатор" - идентификационный номер сертификата;

  • "Дата начала действия" - дата начала действия сертификата;

  • "Дата окончания действия" - дата окончания действия сертификата;

  • "Статус" - статус сертификата в системе: "Активный", "Неактивный", "Истек", "Аннулирован".

    Статус "Активный" обозначает, что клиент может использовать данный сертификат для защиты информации, которую он передает в банк.

    Статус "Неактивный" обозначает, что сертификат создан в системе, но не привязан данному клиенту банка.

    Статус "Истек" обозначает, что время действия сертификата закончилось.

    Статус "Аннулирован" может обозначать, что клиент отказался от сертификата либо что с данным клиентом расторгнут договор на предоставление услуг.

    Статус "Заблокирован" означает, что сертификат заблокирован по запросу клиента (утеря, компрометация и пр.). Сертификат с таким статусом не может быть использован клиентом для работы в системе.

Вы можете найти созданный сертификат с помощью Фильтра. Чтобы осуществить поиск, щелкните по кнопке Показать фильтр и правильно заполните поля фильтра:

  • "Дата начала действия" - дата начала действия сертификата;

  • "Дата окончания действия" - дата окончания действия сертификата;

  • "Статус" - статус сертификата в системе: "Активный", "Неактивный", "Истек", "Аннулирован".

Также Вы можете фильтровать список по любой комбинации полей. Например, можно ввести в поля фильтра только статус сертификата.

После того как параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас сертификатов.

Пакет документов

При необходимости назначьте пользователю пакет услуг, который определяет перечень шаблонов документов (договоров), которые печатаются и подписываются клиентом при подключении к системе. Пакет документов формируется в пункте главного меню Сервис - Пакет шаблонов.

Назначение пакета осуществляется через пункт меню Пакет документов. При его выборе на экран выводится список существующих в системе пакетов услуг, из которых требуется выбрать тот, который назначается данному клиенту.

Щелкнув по кнопке Печать договора, Вы можете распечатать пакет договоров для подключения клиента к системе в соответствии с назначенным пользователю пакетом услуг.

Блокировка и разблокировка клиентов

Учетные записи клиентов можно блокировать. Это позволяет запретить клиенту доступ в систему (например, из-за того, что данная услуга не была своевременно оплачена или пин-конверт был потерян), не удаляя его учетную запись.

Блокировка выполняется следующим образом:

  1. В главном меню системы щелкните по пункту Клиенты. На экран будет выведен список всех зарегистрированных в системе клиентов.

  2. В списке выберите тех клиентов, учетные записи которых будут заблокированы, установив флажки, расположенные рядом с их именами.

  3. Щелкните по расположенной над списком клиентов кнопке Заблокировать.

Разблокировка учетных записей выполняется точно также, только отмечаются заблокированные учетные записи и выбирается кнопка Разблокировать.

Заблокированных клиентов в списке можно отфильтровать с помощью Фильтра/поиска клиентов. Для этого в фильтре надо задать статус клиента "Заблокирован", и система покажет в списке только заблокированных клиентов.

Визуально отличить заблокированного клиента можно по пиктограмме , которая появляется рядом с фамилией заблокированного клиента банка.
