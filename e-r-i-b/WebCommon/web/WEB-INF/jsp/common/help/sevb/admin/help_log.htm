Пункт меню Сервис позволяет Вам выполнять различные сервисные операции, например, просматривать журналы, в которых регистрируются действия сотрудников и клиентов банка, архивировать журналы, менять свой пароль для входа в систему и т.д.

Журнал действий пользователей

Фильтр операций

Пункт меню Журнал действий пользователей - протокол действий пользователей, в который занесены все операции, выполненные пользователями системы.

Данный пункт меню используется для контроля за действиями клиентов и сотрудников банка в системе. Он предназначен для администраторов и сотрудников, которые занимаются аудитом.

В журнале действий пользователей по каждой совершенной операции можно просмотреть следующую информацию:

  • "Номер записи" - порядковый номер записи.

  • "Дату и время" - дата и время совершения операции.

  • "Приложение" - отображается приложение клиента банка (для клиента) или приложение АРМ-сотрудника (для сотрудника).

  • "Идентификатор пользователя" - идентификационный номер пользователя.

  • "ФИО" - фамилия, имя, отчество пользователя системы.

  • "Подразделение" - подразделение, за которым закреплен определенный пользователь.

  • "Наименование операции" - название операции, выполненной пользователем.

  • "Параметры операции" - параметры операции, выполненной клиентом или сотрудником банка.

  • "IP-адрес" - IP-адрес пользователя системы.

  • "Идентификатор сессии" - уникальный идентификатор сессии пользователя.

  • "Тип сообщения" - в системе предусмотрены следующие типы сообщений: "информационное" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "безопасность" (сообщение подсистемы безопасности о том, что пользователь совершил операцию, доступ к которой ему запрещен), "ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

Фильтр операций

Для просмотра списка всех интересующих Вас операций, выполненных пользователем системы, щелкните по кнопке Показать фильтр и заполните поля фильтра:

  • "ФИО" - поиск по фамилии, имени и отчеству пользователя;

  • "Период" - поиск по дате и времени, за которое были совершены операции в системе;

  • "Приложение" - поиск по приложению, в котором пользователем была совершена операция;

  • "Идентификатор пользователя" - поиск по идентификационному номеру пользователя в системе;

  • "Подразделение" - поиск по номеру подразделения, за которым закреплен пользователь системы. Для того чтобы осуществить поиск по подразделению, щелкните по кнопке (...), откроется справочник, в котором выберите интересующее Вас подразделение системы "Сбербанк ОнЛ@йн".

  • "IP-адрес" - поиск по IP-адресу пользователя;

  • "Наименование операции" - поиск по названию операции;

  • "Тип сообщения " - поиск по типу сообщения: "информационное" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "безопасность" (сообщение подсистемы безопасности о том, что пользователь совершил операцию, доступ к которой ему запрещен), "ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

  • "Максимальное количество записей " - настройка отображения, определяющая, сколько записей будет выведено на страницу. Если все требуемые записи не помещаются на одной странице, то будет произведена разбивка по страницам с указанным количеством записей.

В фильтре автоматически задается период выполненных действий за прошедшие сутки, но сотрудник банка может задавать любой интересующий его период, за который были выполнены действия пользователя системы.

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв фамилии, имени или отчества пользователя.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать всех пользователей с фамилией на букву "А" из одного подразделения банка.

После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас операций пользователей.

Если Вам необходимо выгрузить определенные записи журнала в файл, щелкните по кнопке Выгрузить , откроется окно, в котором нужно указать операцию Сохранить на диск и затем выбрать или ввести имя файла для сохранения.

Журнал системных действий

Фильтр операций

Журнал системных действий используется для фиксации ошибок и сбоев, произошедших при работе пользователей в системе, запросов, пришедших от пользователей, задач, запущенных по расписанию, для отслеживания попыток взлома системы, а также содержит отладочную информацию. Он предназначен для системных администраторов и сотрудников банка, которые занимаются аудитом.

В журнале системных действий можно просмотреть следующую информацию:

  • "Номер записи" - порядковый номер записи;

  • "Дату и время" - дата и время совершения операции.

  • "Тип сообщения" - тип сообщения: "информация" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "диагностика" (сообщение, предупреждающее о возможности сбоя, ошибки в системе), "ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

  • "Приложение" - приложение, в котором совершена операция: АРМ-сотрудника, клиента, обработка оповещений и др.

  • "Идентификатор пользователя" - идентификационный номер пользователя системы;

  • "ФИО" - фамилия, имя, отчество пользователя системы;

  • "IP-адрес" - IP-адрес пользователя системы;

  • "Идентификатор сессии" - уникальный идентификатор сессии пользователя;

  • "Сообщение" - текст сообщения системы.

  • "ФИО" - поиск по фамилии, имени и отчеству пользователя системы;

  • "Период" - поиск по дате и времени, за которое были выполнены системные действия;

  • "Идентификатор ошибки" - поиск по идентификатору ошибки, случившейся в результате работы пользователей в системе. Данный идентификатор отображается пользователю на экране при возникновении ошибки.

  • "Слово в тексте" - поиск по слову в тексте сообщения системы;

  • "Тип сообщения" - поиск по типу сообщения системы: "информация" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "диагностика" (сообщение, предупреждающее о возможности сбоя, ошибки в системе), "ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

  • "Максимальное количество записей " - настройка отображения, определяющая, сколько записей будет выведено на страницу. Если все требуемые записи не помещаются на одной странице, то будет произведена разбивка по страницам с указанным количеством записей.

В фильтре автоматически задается период выполненных действий за прошедшие сутки, но Вы можете задать любой интересующий Вас период, за который были выполнены системные действия.

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв фамилии, имени или отчества пользователя.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать сообщения по приложению, в котором совершена операция, и по слову в тексте сообщения.

После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас операций, совершенных системой.

Если Вам необходимо выгрузить определенные записи журнала в файл, щелкните по кнопке Выгрузить, откроется окно, в котором нужно указать операцию Сохранить на диск и затем выбрать или ввести имя файла для сохранения.

Журнал сообщений

Фильтр сообщений

В пункте меню Журнал сообщений (журнал взаимодействий с внешними системами) отображаются сообщения между системой ИКФЛ «Сбербанк ОнЛ@йн» и внешними системами RS-Retail, ИСППН "Город", АС "Стоп-Лист" и ПЦ (процессинговый центр банка).

В журнале сообщений по каждому сообщению, полученному от RS-Retail, ИСППН "Город", АС "Стоп-Лист" и ПЦ или отправленному внешней системе, можно просмотреть следующую информацию:

  • "Номер записи" - номер сообщения.

  • "Дату и время" - дата и время сообщения.

  • "Приложение" - приложение, которое отправило сообщение во внешнюю систему.

  • "Систему" - система, которой адресовано сообщение (исходящее), или система, от которой получено сообщение (входящее).

  • "Подразделение" - подразделение, доступное пользователю.

  • "Тип запроса" - вид XML-сообщения, которое отправляется из системы ИКФЛ в RS-Retail или другую внешнюю систему.

  • "Тип ответа" - вид XML-сообщения, которое отправил RS-Retail или другая внешняя система на запрос.

  • "Сообщения" - содержание сообщения системы. Для того чтобы просмотреть содержание сообщения, щелкните по кнопке <...>, откроется страница просмотра сообщения, на которой система покажет Вам запрос ИКФЛ «Сбербанк ОнЛ@йн» во внешнюю систему и ответ на запрос.

Фильтр/поиск сообщений

Для просмотра списка всех интересующих Вас сообщений щелкните по кнопке Показать фильтр и заполните поля фильтра:

  • "Тип запроса" - поиск по виду (наименованию) XML-сообщения, которое было отправлено из системы ИКФЛ «Сбербанк ОнЛ@йн» в RS-Retail или другую внешнюю систему. Тип отражает, что за сообщение было отправлено или получено шлюзом системы: запрос на получение остатка денежных средств на счете, запрос и получение остатка денежных средств на пластиковой карте, запрос и получение комиссии по платежной операции и т.д.

  • "Тип ответа" - поиск по виду (наименованию) XML-сообщения, которое было получено системой ИКФЛ «Сбербанк ОнЛ@йн» от RS-Retail или от другой внешней системы.

  • "Период" - поиск по дате и времени, за которое были отправлены и получены сообщения системой;

  • "Подразделение" - поиск по подразделению, в котором произошло действие. Для того чтобы осуществить поиск по подразделению, щелкните по кнопке (...), откроется справочник, в котором выберите интересующее Вас подразделение системы "Сбербанк ОнЛ@йн".

  • "Слово в запросе" - поиск по слову в запросе;

  • "Слово в ответе" - поиск по слову в ответе;

  • "Система" - система, которой адресовано сообщение (исходящее), или система, от которой получено сообщение (входящее).

  • "Приложение" - приложение, которое отправило сообщение во внешнюю систему.

  • "Максимальное количество записей " - настройка отображения, определяющая, сколько записей будет выведено на страницу. Если все требуемые записи не помещаются на одной странице, то будет произведена разбивка по страницам с указанным количеством записей.

В фильтре автоматически задается период выполненных действий за прошедшие сутки, но Вы можете задавать любой интересующий Вас период, за который были получены и отправлены сообщения.

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв слова в запросе.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать сообщения по приложению, в котором совершена операция, и по слову в ответе.

После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас сообщений.

Если Вам необходимо выгрузить определенные записи журнала в файл, щелкните по кнопке Выгрузить, откроется окно, в котором нужно указать операцию Сохранить на диск и затем выбрать или ввести имя файла для сохранения.

Архивация журналов

Данный пункт меню предназначен для обеспечения быстродействия системы, а также для уменьшения размера базы данных.

С помощью пункта меню Архивация журналов можно выгрузить из базы ИКФЛ «Сбербанк ОнЛ@йн» протокол аудита за определенный период в архив (протокол действий пользователей, протокол системных действий и протокол сообщений), можно загрузить в систему записи протокола из файла-архива. Данный пункт предназначен для системных администраторов и сотрудников банка, которые занимаются аудитом.

Для того чтобы выгрузить протокол из базы ИКФЛ «Сбербанк ОнЛ@йн», в поле "Вид журнала" выберите из списка журнал, который хотите заархивировать, в поле "Период" укажите период, записи за который хотите поместить в архив. Затем щелкните по кнопке В архив и сохраните xml-файл на диск. В результате записи за указанный период будут удалены из журнала и помещены в архив.

Для того чтобы загрузить журнал в базу ИКФЛ «Сбербанк ОнЛ@йн», в поле «Разархивация» щелкните по кнопке Обзор, выберите интересующий архив записей журнала и нажмите кнопку Из архива . В итоге протоколы загрузятся в базу.

Сменить пароль

Чтобы изменить пароль для входа в систему, в поле "Введите новый пароль" впишите Ваш новый пароль, потом подтвердите новый пароль, заполнив поле "Повторите новый пароль" той же комбинацией символов. Затем щелкните по кнопке Изменить , откроется окно, в котором будет написано "Пароль на вход в систему изменен". В результате при следующем входе в систему Вы должны будете указать в поле "пароль" Ваш новый пароль.

Время приема документов

Пункт меню Время приема документов предназначен для настройки режима приема платежных документов от клиентов банка. В данном пункте меню задаются границы времени приема документов в течение дня, которые определяют, каким днем будут приниматься платежные документы. Например, если для перевода между счетами Вы укажете время приема документов 16:00, то документы, полученные позже 16:00, будут обрабатываться системой следующим операционным днем.

По умолчанию режим приема документов задан 17:00. Вы можете посмотреть и при необходимости изменить настройки. Для изменения настройки выберите документ, введите в поле новое значение, время должно быть в формате ЧЧ:ММ, и щелкните по кнопке Изменить . В результате время приема выбранного Вами документа будет изменено в системе.

Формы платежей

Добавление новой формы платежа

В пункте меню Формы платежей содержится список форм платежей, из которого Вы можете добавить клиенту различные виды платежей.

Добавление новой формы платежа

Создать новый платеж можно на основе одного из уже существующих платежей.

Надо выбрать среди имеющихся платежей наиболее близкий по составу полей. Затем следует выгрузить файлы выбранного платежа на диск.

Чтобы выгрузить новый платеж, надо:

  • выбрать платеж в списке, установив галочку напротив интересующего Вас платежа;

  • щелкнуть по кнопке Выгрузить . Откроется страница, на которой находятся ссылки на файлы, которые нужно выгрузить. Для того чтобы выгрузить файлы, щелкните по каждой ссылке и нажмите кнопку Сохранить .

  • переименовать выгруженные файлы;

  • внести необходимые изменения;

  • затем загрузить новый платеж.

Чтобы загрузить новый платеж, нужно:

  • выбрать раздел главного меню Сервис, пункт Формы платежей,

  • щелкнуть по кнопке Добавить ,

  • указать файлы на диске.

Для того чтобы платеж появился в клиентском приложении, добавьте его на страницу списка платежей.

Проверка АСП

Пункт меню Проверка АСП (проверка аналога собственноручной подписи клиента) может быть использован для разрешения конфликтных ситуаций с клиентами. В нем можно просмотреть по каждой платежной операции, совершенной клиентом, какие поля были подписаны и проконтролировать совпадение АСП.

Для того чтобы проверить АСП клиента, в поле "Идентификатор платежа/заявки" необходимо ввести идентификационный номер платежа или заявки и щелкнуть по кнопке Проверить . В итоге система покажет результаты первоначальной и повторной проверки.

Справочник ошибок

Фильтр/поиск сообщений об ошибках

Пункт меню Справочник ошибок используется для просмотра, создания и изменения сообщений системы об ошибках пользователей. Он предназначен для обработки сообщений, полученных от внешней системы RS-Retail и других систем об ошибках клиентов. В данном пункте задается соответствие между текстом сообщения из внешней системы и интерпретацией его в системе ИКФЛ «Сбербанк ОнЛ@йн». Например, если при ошибке из RS-Retail получено сообщение "(679)", то клиенту придет сообщение "Превышен размер лимита".

Для того чтобы создать новое сообщение об ошибке, щелкните по кнопке Создать , откроется форма редактирования сообщения. В данной форме в поле "Тип ошибки" по умолчанию задается значение "Клиентская", в поле "Система" выберите систему, из которой получено сообщение об ошибке, в поле "Регулярное выражение" укажите регулярное выражение ошибки, присланной из внешней системы, в поле "Сообщение в ИКФЛ" введите сообщение, которое отправляется клиенту банка при ошибке в системе. Затем нажмите кнопку Сохранить , и новое сообщение об ошибке появится в списке.

Для изменения сообщения установите галочку напротив интересующего сообщения и щелкните по кнопке Изменить , система покажет Вам форму редактирования сообщения. В данной форме измените значение в интересующем Вас поле и щелкните по кнопке Сохранить . Если Вы не хотите менять сообщение об ошибке, то щелкните по кнопке Отменить .

Если Вы хотите удалить какое-либо сообщение об ошибке клиента, установите галочку напротив интересующего Вас сообщения и щелкните по кнопке Удалить . Система попросит Вас подтвердить операцию, Вы подтверждаете, и выбранная ошибка будет удалена из списка.

Фильтр/поиск сообщений об ошибках

Для просмотра списка всех интересующих Вас сообщений об ошибке пользователя системы щелкните по кнопке Показать фильтр и заполните поля фильтра:

  • "Регулярное выражение" - поиск по регулярному выражению ошибки;

  • "Тип ошибки" - поиск по типу ошибки пользователя;

  • "Система" - поиск по названию системы, в которой допущена ошибка.

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых регулярного выражения ошибки.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать сообщения по системе, в которой совершена ошибка, и по типу ошибки.

После того как искомые параметры заданы, щелкните по кнопке Сохранить , и система покажет список интересующих Вас сообщений.
