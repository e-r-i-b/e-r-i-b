Параметры системы

С помощью пункта меню Настройки задаются параметры системы. Вы можете задать настройки, определяющие функционирование системы.

Клиенты и сотрудники

Пункт меню Клиенты и сотрудники предназначен для управления настройками системы, которые регулируют работу клиентов и сотрудников банка в системе. В данном пункте меню Вы можете управлять следующими настройками системы:

  1. Разрешенные символы для пароля

    Данная настройка предназначена для управления символами, которые создаются в системе для ввода пароля. Чтобы изменить допустимые символы, используемые для создания паролей, в поле "Разрешенные символы для пароля" выберите из списка нужное значение и нажмите на кнопку Сохранить. Например, если Вы выбираете значение "Только цифры", то при создании пароля утилита печати генерирует пароль, состоящий только из цифр. После сохранения настройка будет изменена в системе и вступит в действие.

    Примечание: помимо настройки пароля, есть возможность контроля за качеством пароля в системе в конфигурационных файлах, т.е. администратор системы может задать более строгие критерии для создания пароля.

  2. Разрешенные символы для логина

    Данная настройка используется для изменения допустимых символов, используемых при создании логинов. Чтобы изменить разрешенные символы для логина, в поле "Разрешенные символы для логина" выберите из списка необходимое значение и нажмите на кнопку Сохранить. Например, если Вы выбираете значение "Цифры и заглавные буквы", то при создании логина система будет генерировать логины для сотрудников и клиентов, состоящие только из цифр и заглавных букв. После сохранения настройка будет изменена в системе и вступит в действие.

  3. Настройка длины пароля

    Данная настройка предназначена для определения, какой длины будут создаваемые утилитой печати пароли, и для общего ограничения длины паролей, вводимых клиентами и сотрудниками банка. Если Вы хотите изменить длину создаваемых паролей, введите количество допустимых символов в поле "Длина пароля" и щелкните по кнопке Сохранить. После сохранения настройка вступит в действие.

    Примечание: помимо настройки пароля, есть возможность контроля за качеством пароля в системе в конфигурационных файлах, т.е. администратор системы может задать более строгие критерии для создания пароля.

  4. Количество попыток при регистрации

    Данная настройка используется для изменения количества попыток для ввода пароля до автоматической блокировки доступа в систему. Чтобы изменить количество попыток, введите в поле "Количество попыток (регистрация)" число попыток и щелкните по кнопке Сохранить. После сохранения настройка вступит в действие.

  5. Период блокировки

    Данная настройка предназначена для изменения времени, на которое блокируется доступ к системе при многократном неверном вводе пароля. Для изменения периода блокировки введите в поле "Период блокировки" числовое значение (период блокировки исчисляется в секундах) и щелкните по кнопке Сохранить.

    Если Вы меняете только одну настройку, то после ее изменения нажимаете по кнопке Сохранить, если Вы меняете сразу несколько настроек, то Вы вводите изменение в каждое изменяемое поле и потом нажимаете Сохранить.

    Эти параметры действуют для всех зарегистрированных сотрудников и клиентов и вступают в силу сразу же после сохранения изменений.

Права доступа сотрудника

Пункт Права доступа сотрудника позволяет задать схему прав, которая автоматически назначается при создании нового сотрудника банка в системе.

В данном пункте меню в поле "Схема прав сотрудника" выберите из списка схему прав, которая будет по умолчанию назначена сотрудникам банка, после чего щелкните по кнопке Сохранить. Настройка вступит в действие после сохранения.

Права доступа клиента

Пункт Права доступа клиента позволяет задать способ аутентификации клиентов банка по умолчанию, для каждого способа задается схема прав, которая автоматически назначается при создании нового клиента банка в системе, а также способ подтверждения входа и способ подтверждения операций клиентами банка.

Упрощенный доступ

Данный тип доступа используется при работе клиентов банка в системе ИКФЛ без сертификата. Для того чтобы задать упрощенный доступ для создаваемых в системе клиентов банка, установите флажок напротив строки "Упрощенный доступ". При данном типе доступа Вы можете управлять следующими настройками системы:

  1. Подтверждение входа

    Настройка Подтверждение входа предназначена для указания того, каким образом клиент банка будет подтверждать вход в систему. Для изменения данной настройки выберите интересующее Вас значение в поле "Подтверждение входа":

    • "Без подтверждения" - клиент банка входит в систему только при помощи логина и пароля;

    • "Одноразовый пароль" - при входе в систему клиент банка должен ввести не только логин, пароль, а также требуемый системой ключ из карты ключей.

    • "SMS" - при входе в систему клиенту банка после ввода логина и пароля, необходимо вписать SMS-пароль, который приходит клиенту банка в виде SMS на номер, указанный в анкете.

      Примечание: SMS-пароль – это одноразовый пароль, формируемый системой ИКФЛ.

  2. Подтверждение операций

    В настройке Подтверждение операций укажите, каким способом в системе будет подтверждаться совершение операций клиентом банка, например, одноразовым паролем с карты ключей, SMS-паролем и т.д. Для изменения данной настройки выберите в поле "Подтверждение операций" одно из значений:

    • "Без подтверждения" - клиент банка совершает операции в системе без дополнительного подтверждения;

    • "Одноразовый пароль" - при совершении операции клиент банка должен ввести требуемый системой ключ из карты ключей;

    • "SMS" - при совершении операций клиенту банка нужно будет ввести SMS-пароль, который получит клиент банка в виде SMS на номер, указанный в анкете.

  3. Схема прав

    Настройка Схема прав предназначена для указания схемы прав, назначаемой системой для упрощенного доступа по умолчанию. Для изменения данной настройки в поле "Cхема прав" задайте из списка интересующую Вас схему прав и нажмите на кнопку Сохранить.

После заполнения всех полей щелкните по кнопке Сохранить.

Защищенный доступ

Данный тип доступа предназначен для входа клиентов банка в систему в безопасном режиме по сертификату СКЗИ, для подтверждения операций и подписания документов электронно-цифровой подписью (ЭЦП). Для того чтобы задать защищенный доступ для создаваемых в системе клиентов банка, установите флажок напротив строки "Защищенный доступ". При данном типе прав доступа Вы можете управлять следующими настройками системы:

  1. Подтверждение входа

    Данная настройка определяет способ подтверждения входа клиентов банка в систему. По умолчанию указывается значение "ЭЦП" (электронно-цифровая подпись).

  2. Подтверждение операций

    Данная настройка определяет способ подтверждения операций клиентов банка в систему. По умолчанию указывается значение "ЭЦП" (электронно-цифровая подпись).

  3. Схема прав

    Данная настройка регулирует, какая схема прав будет назначаться клиентам банка по умолчанию при аутентификации по защищенному доступу. Чтобы задать схему прав доступа для клиентов банка, в поле "Схема прав" задайте из списка интересующую Вас схему прав и нажмите на кнопку Сохранить.

    После заполнения всех полей щелкните по кнопке Сохранить, и настройка вступит в действие.

    Карты ключей

    В пункте меню Карты ключей можно указать, сколько карт ключей будет генерироваться одновременно, количество ключей на каждой из создаваемых карт, набор символов, из которых будут состоять создаваемые ключи, длину ключей в карте. Вы можете управлять следующими настройками генерации карт ключей в системе:

    1. Создание карт

      Данная настройка предназначена для управления количеством карт и ключей на карте при создании карт ключей. Для изменения настроек при создании карт Вам нужно:

      В параметре "Создание карт" укажите количество карт, создаваемых за одну процедуру генерации, для этого введите в данное поле числовое значение от 1 до 100 карт ключей. Также в данной настройке введите числовое значение от 1 до 500, определяющее количество ключей на карте, и щелкните по кнопке Сохранить. После сохранения настройка вступит в действие.

    2. Разрешенные символы

      Данная настройка позволяет управлять символами, которые используются системой при генерации карт ключей. Для изменения разрешенных символов в поле "Разрешенные символы" выберите из списка интересующее Вас значение и щелкните по кнопке Сохранить. Настройка вступит в действие после сохранения.

    3. Длина ключа

      Данная настройка позволяет Вам регулировать длину ключей на карте, генерируемых системой. Для изменения длины пароля в поле "Длина ключа" введите числовое значение от 1 до 20, затем щелкните по кнопке Сохранить.

      Если Вы меняете только одну настройку, то после ее изменения нажимаете по кнопке Сохранить, если Вы меняете сразу несколько настроек, то Вы вводите изменение в каждое изменяемое поле и потом нажимаете Сохранить.

    Указанные здесь настройки генерации карт ключей будут использоваться по умолчанию, но при генерации карт в пункте главного меню Карты ключей - Генерация карт ключей их можно изменить.

    SMS-пароли

    Пункт меню SMS-пароли предназначен для управления настройками SMS-паролей, используемых для подтверждения входа в систему и для подтверждения операций клиентами банка.

    В системе ИКФЛ Вы можете изменить и сохранить следующие параметры формирования SMS-паролей:

    1. Длина сообщения

      Данная настройка предназначена для определения длины SMS-сообщения, которое будет приходить клиенту банка на номер мобильного телефона, указанный в анкете. Чтобы изменить длину пароля, в поле «Длина сообщения» укажите количество символов для формирования SMS-пароля.

    2. Количество попыток ввода SMS-пароля

      Данная настройка используется для изменения количества попыток ввода SMS-пароля. Для изменения количества попыток в поле «Количество попыток ввода» укажите, сколько раз клиент может ввести SMS-пароль до автоматической блокировки.

    3. Время жизни пароля

      Данная настройка предназначена для изменения времени, в течение которого клиент сможет ввести пароль для подтверждения входа или операций в системе. Чтобы изменить настройку, в поле «Время жизни (в секундах)» введите время действия SMS-пароля.

    4. Используемые символы

      В поле «Используемые символы» можно выбрать из списка символы, которые будут использоваться при создании SMS-пароля.

    После заполнения всех полей щелкните по кнопке Сохранить , и настройки вступят в действие.

    Справочники CONTACT

    Пункт меню Справочники CONTACT предназначен для указания пути к каталогу, из которого будут реплицироваться справочники системы CONTACT.

    Для изменения настройки в поле "Каталог для загрузки справочников CONTACT" введите путь к каталогу, из которого будут реплицироваться справочники, и нажмите на кнопку Сохранить. Если Вы хотите отменить несохраненные изменения, щелкните по кнопке Очистить.

    Справочники

    Пункт меню Справочники предназначен для указания внешней системы, из которой должны загружаться справочники в систему ИКФЛ.

    Для того чтобы задать внешнюю систему, в поле "Внешняя система для загрузки справочников" щелкните по кнопке (...). Откроется справочник внешних систем, в котором установите флажок напротив выбранной внешней системы и щелкните по кнопке Выбрать. На данной странице Вы можете воспользоваться фильтром/поиском внешних систем.

    Фильтр/поиск внешних систем

    Для поиска интересующих Вас внешних систем можно воспользоваться фильтром. Для этого щелкните по кнопке Показать фильтр и задайте критерии поиска:

    • "Идентификатор" - поиск по уникальному идентификатору внешней системы;

    • "Наименование" - поиск по наименованию шлюза;

    • "Адрес шлюза" - поиск по адресу шлюза.

    Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв наименования внешней системы.

    Можно отфильтровать список по любой комбинации полей. Например, можно отфильтровать все внешние системы по наименованию и адресу шлюза.

    После того как критерии поиска заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас внешних систем.

    После выбора внешней системы нажмите на кнопку Сохранить, и настройка вступит в действие.

    В данном пункте меню можно настроить виды периодичности оповещений, которые будут доступны сотруднику банка для подписки. В списке расписаний по каждому оповещению есть возможность настроить параметры, задающие его периодичность.

    Список расписаний сотрудников

    В списке расписаний по каждому оповещению отображаются:

    1. Название оповещения;

    2. Периодичность срабатывания оповещения;

    3. Выражение.

    Вид оповещения может быть одним из следующих:

    • "Блокировка пользователя" - через заданный промежуток времени система проверяет, изменился ли статус пользователей на "Заблокирован", если статус изменился, то сотруднику банка придет сообщение о блокировке пользователей.

    • "Окончание срока действия сертификата" - через заданный промежуток времени система проверяет срок действия сертификатов клиентов банка, если срок действия истек, то сотруднику банка придет оповещение.

    • "Блокировка карты ключей" - через заданный промежуток времени система проверяет, изменился ли статус карт ключей клиентов банка на "Заблокирована", если статус изменился, то сотруднику банка придет сообщение.

    • "Совершение платежной операции или ввод заявки" - через заданный промежуток времени система проверяет количество совершенных платежей или ввод заявок клиентами банка, если кто-либо из клиентов совершил операцию, то сотруднику банка придет сообщение.

    • "Ошибки заданного уровня важности" - через определенный промежуток времени система проверяет, произошли ли ошибки заданного уровня важности, если ошибки произошли, то сотруднику банка придет оповещение.

    Добавление расписания

    В список расписаний Вы можете добавить новое расписание. Для этого щелкните по кнопке Добавить, откроется форма создания рассылки, в которой в поле "Рассылка" выберите из справочника интересующий Вас вид рассылки сообщений, в поле "Выражение" введите то значение, которое в системе соответствует данной рассылке, в поле "Описание" введите содержание описания новой рассылки и щелкните по кнопке Сохранить, и созданная Вами рассылка появится в списке расписаний. Подробную информацию по созданию и изменению регулярного выражения смотрите в пользовательской документации.

    Редактирование расписания

    Список расписаний также можно отредактировать. Для этого установите галочку напротив интересующего Вас вида рассылки, нажмите кнопку Редактировать. Система покажет Вам форму редактирования рассылки, в которой Вы можете изменить значение в поле "Выражение" и в поле "Описание". После изменения параметров нажмите на кнопку Сохранить, и данная рассылка появится в списке рассылок в измененном виде.

    Удаление расписания

    Если Вы хотите удалить расписание оповещения, установите галочку напротив интересующей Вас рассылки и щелкните по кнопке Удалить. Система попросит Вас подтвердить удаление выбранной рассылки, после подтверждения данная рассылка будет удалена из списка расписаний.

    Добавление расписания
    Редактирование расписания
    Удаление расписания

    В данном пункте меню можно настроить виды периодичности оповещений, которые будут рассылаться клиентам банка. В списке расписаний по каждому оповещению есть возможность настроить параметры, задающие его периодичность.

    Список расписаний клиентов

    В списке расписаний по каждому оповещению отображаются:

    1. Вид настройки;

    2. Периодичность срабатывания оповещения;

    3. Название оповещения.

    Вид оповещения может быть одним из следующих:

    • "Окончание срока действия сертификата" - система через заданный промежуток времени проверяет срок действия сертификатов клиентов банка, если срок истек, то клиентам придет сообщение на E-mail или SMS-сообщение.

    • "Произведена операция по счету/карте" - система через заданный промежуток времени проверяет совершение операций по счетам клиентов, если операции произведены, то клиентам банка придет сообщение на E-mail или SMS-сообщение.

    • "Блокировка карты ключей" - система через заданный промежуток времени проверяет изменение статуса карт ключей клиентов банка на "Заблокирована", если статус карт изменился, то клиентам придет сообщение на E-mail или SMS-сообщение.

    • "Приближающееся окончание срока действия депозита" - система через заданный промежуток времени проверяет приближение даты окончания срока действия депозитов, при приближении срока клиентам придет оповещение на E-mail или SMS-сообщение.

    • "Выписка по счету за прошедший период" - система с заданной периодичностью высылает клиентам банка на E-mail оповещение, содержащее выписку по счетам клиентов банка за прошедший период.

    • "Извещение об остатке на счете" - система с заданной периодичностью высылает клиентам банка на E-mail или в виде SMS оповещение, содержащее извещение о текущих остатках средств на счетах клиентов банка.

    • "Уменьшение остатка на счете ниже заданного минимума" - система через заданный промежуток времени проверяет остаток на счетах клиентов банка, если он меньше заданного уровня, то клиентам приходит оповещение на E-mail или SMS-сообщение.

    • "Вход пользователя в систему" - система через заданный промежуток времени проверяет вход пользователей в систему, если клиенты банка заходили в систему, то клиентам придет оповещение на E-mail или SMS-сообщение.

    • "Зачисление средств на счет" - система через заданный промежуток времени проверяет зачисление средств на счета клиентов, если зачисление произошло, то клиентам банка приходит оповещение на E-mail или SMS-сообщение.

    • "Блокировка пользователя" - система через заданный промежуток времени проверяет статус клиентов банка, если статус изменился на "Заблокирован", то клиентам придет оповещение на E-mail или SMS-сообщение.

    • "Изменение статуса платежа или заявки" - через заданный промежуток времени система проверяет изменение статусов платежей и заявок клиентов банка, при изменении состояния платежей и заявок клиентам придет оповещение на E-mail или SMS-сообщение.

    • "Изменение остатка на счете" - система через заданный промежуток времени проверяет изменение остатка на счетах в результате проведения операций клиентами банка, при изменении остатка на счетах клиентам придет оповещение на E-mail или SMS-сообщение.

    Добавление расписания

    В список расписаний Вы можете добавить новое расписание. Для этого щелкните по кнопке Добавить и в поле "Рассылка" выберите из справочника интересующий Вас вид рассылки сообщений, в поле "Выражение" введите то значение, которое в системе соответствует данной рассылке, в поле "Описание" введите содержание описания новой рассылки и щелкните по кнопке Сохранить. Подробную информацию по созданию и изменению регулярного выражения смотрите в пользовательской документации.

    Редактирование расписания

    Список расписаний также можно отредактировать. Для этого установите галочку напротив интересующего Вас сообщения, нажмите кнопку Редактировать. Система покажет Вам параметры рассылки, среди которых Вы можете изменить значение в поле "Выражение" и в поле "Описание". После изменения параметров нажмите кнопку Сохранить и данная рассылка появится в списке рассылок в измененном виде.

    Удаление расписания

    Если Вы хотите удалить расписание сообщения, установите галочку напротив интересующей Вас рассылки и щелкните по кнопке Удалить. Система попросит Вас подтвердить удаление выбранной рассылки, после подтверждения данная рассылка будет удалена из списка расписаний.

    Текущий стиль

    Пункт меню Текущий стиль предназначен для смены текущего (активного) стиля приложения и изменения пути к каталогу глобальных настроек (общих стилей).

    В данном пункте меню Вы можете управлять следующими настройками текущего стиля:

    1. Текущий стиль клиентского приложения

      Для того чтобы изменить текущий стиль клиентского приложения, в поле «Клиентское приложение» выберите из выпадающего списка интересующий Вас стиль приложения.

    2. Текущий стиль АРМ сотрудника банка

      Для того чтобы изменить текущий стиль приложения АРМ сотрудника, в поле «АРМ Сотрудника банка» выберите из выпадающего списка интересующий Вас стиль приложения.

    3. Путь к каталогу общих стилей

      Данная настройка предназначена для определения пути к каталогу, в котором хранятся стили приложений. Для изменения пути к каталогу глобальных настроек (общих стилей) в поле «Каталог общих стилей» щелкните по кнопке Обзор и укажите путь к каталогу.

      Далее необходимо проверить, правильно ли задан путь, для этого щелкните по кнопке Тест. В результате откроется страница, на которой Вы увидите надпись «Если Вы видите логотип банка, значит путь к каталогу общих стилей корректный» и логотип банка.

    После того как все параметры заданы, щелкните по кнопке Сохранить. Если Вам нужно отменить изменения, нажмите на кнопку Очистить.

    Примечание: настройки текущего стиля вступят в действие только после перезапуска приложения.

    Управление стилями

    Пункт меню Управление стилями используется для создания, редактирования и удаления стилей приложений.

    В данном пункте меню открывается страница доступных стилей приложений, на которой для каждого стиля отображается его название, тип (системный, пользовательский), активность (т.е. в каком приложении, АРМ сотрудника или клиентском приложении, используется данный стиль, если стиль неактивный, то поле пустое), и Url.

    Создание стиля

    Для того чтобы создать новый стиль приложения, щелкните по кнопке Добавить, откроется страница редактирования, на данной странице в поле «Название» впишите наименование стиля, в поле «Url» введите путь к каталогу, в котором содержатся файлы нового стиля.

    Вы можете проверить корректность отображения стиля, для этого щелкните по кнопке Проверить стиль, и система покажет Вам макет страницы в новом стиле.

    После того как проверка отображения стиля прошла успешно, щелкните по кнопке Сохранить.

    Для того чтобы созданный стиль стал активным, задайте параметры настройки текущего стиля в пункте меню Настройки - Текущий стиль.

    Редактирование стиля

    Чтобы изменить стиль приложения, установите флажок напротив выбранного стиля и щелкните по кнопке Редактировать. Откроется страница, на которой внесите все необходимые изменения и щелкните по кнопке Сохранить.

    Удаление стиля

    Для удаления одного или нескольких стилей приложений установите флажки напротив выбранных стилей и щелкните по кнопке Удалить.

    Примечание: из системы нельзя удалить текущий (активный) стиль и системные стили.

    Параметры системы

    Пункт меню Параметры системы предназначен для управления настройками, которые задают уровни иерархии подразделений и срок хранения заявок, поступивших от клиентов банка.

    Примечание: данная настройка может быть изменена только встроенным администратором.

    В этом пункте меню Вы можете управлять следующими параметрами системы:

    1. Уровень иерархии подразделений

      Данная настройка предназначена для определения уровней иерархии подразделений. Для изменения уровня иерархии подразделений в поле «Уровень иерархии подразделений» впишите нужное значение. По умолчанию задается второй уровень иерархии, означающий, что в системе ИКФЛ можно завести подразделения на уровне головного подразделения (уровень 1) и на уровне подчиненного подразделения (уровень 2).

    2. Срок хранения заявок

      Данная настройка предназначена для управления сроком хранения заявок в системе. Для изменения срока хранения заявок, поступивших от клиентов банка, в поле «Срок хранения заявок» впишите нужное значение.

    После изменения параметров настройки щелкните по кнопке Сохранить , и настройки вступят в действие.
