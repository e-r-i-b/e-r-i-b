О программе

Пункт меню Сервис позволяет Вам выполнять различные сервисные операции, например, просматривать журналы, в которых регистрируются действия сотрудников и клиентов банка, менять свой пароль для входа в систему, загружать новые формы платежей и т.д.

Журнал действий пользователей

Фильтр операций

Пункт меню Журнал действий пользователей - протокол действий пользователей, в который занесены все операции, выполненные пользователями системы.

Данный пункт меню используется для контроля за действиями клиентов и сотрудников банка в системе. Он предназначен для администраторов и сотрудников, которые занимаются аудитом.

В журнале действий пользователей по каждой совершенной операции можно просмотреть следующую информацию:

  • "Номер записи" - порядковый номер записи.

  • "Дату и время" - дата и время совершения операции.

  • "Приложение" - отображается приложение клиента банка (для клиента) или приложение АРМ сотрудника (для сотрудника).

  • "Идентификатор пользователя" - идентификационный номер пользователя.

  • "ФИО" - фамилия, имя, отчество пользователя системы.

  • "Подразделение" - подразделение, за которым закреплен определенный пользователь.

  • "Наименование операции" - название операции, выполненной пользователем.

  • "Параметры операции" - параметры операции, выполненной клиентом или сотрудником банка.

  • "IP-адрес" - IP-адрес пользователя системы.

  • "Идентификатор сессии" - уникальный идентификатор сессии пользователя.

  • "Тип сообщения" - в системе предусмотрены следующие типы сообщений: "Информационное" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "Безопасность" (сообщение подсистемы безопасности о том, что пользователь совершил операцию, доступ к которой ему запрещен), "Ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

Объем информации, фиксируемой в одной записи протокола, зависит от заданной степени детализации. Уровень детализации задается в пункте меню Сервис – Настройка журнала действий пользователей.

Все записи журнала разбиваются по страницам. По умолчанию на одной странице выводится 20 записей. Количество записей, выводимых на одной странице, можно изменить. Для этого внизу страницы в поле «Записей на странице» выберите из выпадающего списка количество записей, которые должны выводиться на одной странице.

Для перехода к предыдущей или следующей странице журнала щелкните по кнопке Предыдущие 20 или Следующие 20.

Примечание: если все записи журнала поместились на одной странице, то настройка разбивки записей по страницам не отображается.

Фильтр операций

Для просмотра списка всех интересующих Вас операций, выполненных пользователем системы, щелкните по кнопке Показать фильтр и заполните поля фильтра:

  • "ФИО" - поиск по фамилии, имени и отчеству пользователя;

  • "Приложение" - поиск по приложению, в котором пользователем была совершена операция;

  • "Идентификатор пользователя" - поиск по идентификационному номеру пользователя в системе;

  • "Подразделение" - поиск по номеру подразделения, за которым закреплен пользователь системы;

  • "IP-адрес" - поиск по IP-адресу пользователя;

  • "Наименование операции" - поиск по названию операции;

  • "Тип сообщения " - поиск по типу сообщения: "Информационное" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "Безопасность" (сообщение подсистемы безопасности о том, что пользователь совершил операцию, доступ к которой ему запрещен), "Ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

  • "Период" - поиск по дате и времени, за которое были совершены операции в системе;

В фильтре автоматически задается период выполненных действий за прошедшие сутки, но сотрудник банка может задавать любой интересующий его период, за который были выполнены действия пользователя системы.

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв фамилии, имени или отчества пользователя.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать всех пользователей с фамилией на букву "А" из одного подразделения банка.

После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас операций пользователей.

Если Вам необходимо выгрузить определенные записи журнала в файл, щелкните по кнопке Выгрузить, откроется окно, в котором нужно указать операцию Сохранить на диск и затем выбрать или ввести имя файла для сохранения.

Журнал системных действий

Фильтр\поиск операций

Журнал системных действий используется для фиксации ошибок и сбоев, произошедших при работе пользователей в системе, запросов, пришедших от пользователей, задач, запущенных по расписанию, для отслеживания попыток взлома системы, а также содержит отладочную информацию. Он предназначен для системных администраторов и сотрудников банка, которые занимаются аудитом.

В журнале системных действий можно просмотреть следующую информацию:

  • "Номер записи" - порядковый номер записи,

  • "Дату и время" - дата и время совершения операции.

  • "Тип сообщения" - тип сообщения: "Информация" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "Диагностика" (сообщение, предупреждающее о возможности сбоя, ошибки в системе), "Ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

  • "Приложение" - приложение, в котором совершена операция: АРМ сотрудника банка, клиента, обработка оповещений и др.;

  • "Модуль" - модуль, в котором произошло системное действие: ядро, шлюз, обработчик расписаний;

  • "Идентификатор пользователя" - идентификационный номер пользователя системы;

  • "ФИО" - фамилия, имя, отчество пользователя системы;

  • "IP-адрес" - IP-адрес пользователя системы;

  • "Идентификатор сессии" - уникальный идентификатор сессии пользователя;

  • "Сообщение" - текст сообщения системы.

Объем информации, фиксируемой в одной записи протокола, зависит от заданной степени детализации. Уровень детализации задается в пункте меню Сервис – Настройка журнала системных действий.

Все записи журнала разбиваются по страницам. По умолчанию на одной странице выводится 20 записей. Количество записей, выводимых на одной странице, можно изменить. Для этого внизу страницы в поле «Записей на странице» выберите из выпадающего списка количество записей, которые должны выводиться на одной странице.

Для перехода к предыдущей или следующей странице журнала щелкните по кнопке Предыдущие 20 или Следующие 20.

Примечание: если все записи журнала поместились на одной странице, то настройка разбивки записей по страницам не отображается.

Для просмотра списка всех интересующих Вас операций системы щелкните по кнопке Показать фильтр и заполните поля фильтра:

  • "ФИО" - поиск по фамилии, имени и отчеству пользователя системы,

  • "Модуль" - поиск по названию модуля (ядро, шлюз, обработчик расписания), в котором произошло системное действие;

  • "Идентификатор пользователя" - поиск по идентификационному номеру пользователя в системе;

  • "Идентификатор ошибки" - поиск по идентификатору ошибки, случившейся в результате работы пользователей в системе. Данный идентификатор отображается пользователю на экране при возникновении ошибки.

  • "IP-адрес" - поиск по IP-адресу пользователя;

  • "Слово в тексте" - поиск по слову в тексте сообщения системы;

  • "Тип сообщения" - поиск по типу сообщения системы: "Информация" (сообщение о том, что пользователь совершил в системе ту или иную операцию), "Диагностика" (сообщение, предупреждающее о возможности сбоя, ошибки в системе), "Ошибка" (сообщение об операции пользователя, вызвавшей ошибку в системе).

  • "Период" - поиск по дате и времени, за которое были выполнены системные действия;

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв фамилии, имени или отчества пользователя.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать сообщения по приложению, в котором совершена операция, и по слову в тексте сообщения.

После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас операций, совершенных системой.

Если Вам необходимо выгрузить определенные записи журнала в файл, щелкните по кнопке Выгрузить, откроется окно, в котором нужно указать операцию Сохранить на диск и затем выбрать или ввести имя файла для сохранения.

Журнал сообщений

Фильтр сообщений

В пункте меню Журнал сообщений (журнал взаимодействий с внешними системами) отображаются сообщения между системой ИКФЛ и другими внешними системами, например, RS-Retail, RS-Loans и т.д.

В журнале сообщений по каждому сообщению, полученному или отправленному внешней системе, можно просмотреть следующую информацию:

  • "Номер записи" - номер сообщения.

  • "Дата и время" - дата и время сообщения.

  • "Приложение" - приложение, которое отправило сообщение во внешнюю систему.

  • "Система" - система, которой адресовано сообщение (исходящее), или система, от которой получено сообщение (входящее).

  • "Тип запроса" - вид XML-сообщения, которое отправляется из системы ИКФЛ во внешнюю систему.

  • "Тип ответа" - вид XML-сообщения, которое отправила внешняя система на запрос.

  • "Сообщения" - содержание сообщения системы. Для того чтобы просмотреть содержание сообщения, щелкните по кнопке <...>, откроется страница просмотра сообщения, на которой система покажет Вам запрос ИКФЛ во внешнюю систему и ответ на запрос.

Объем информации, фиксируемой в одной записи протокола, зависит от заданной степени детализации. Уровень детализации задается в пункте меню Сервис – Настройка журнала сообщений.

Все записи журнала разбиваются по страницам. По умолчанию на одной странице выводится 20 записей. Количество записей, выводимых на одной странице, можно изменить. Для этого внизу страницы в поле «Записей на странице» выберите из выпадающего списка количество записей, которые должны выводиться на одной странице.

Для перехода к предыдущей или следующей странице журнала щелкните по кнопке Предыдущие 20 или Следующие 20.

Примечание: если все записи журнала поместились на одной странице, то настройка разбивки записей по страницам не отображается.

Фильтр/поиск сообщений

Для просмотра списка всех интересующих Вас сообщений щелкните по кнопке Показать фильтр и заполните поля фильтра:

  • "Тип запроса" - поиск по виду (наименованию) XML-сообщения, которое было отправлено из системы ИКФЛ во внешнюю систему. Тип отражает, что за сообщение было отправлено или получено шлюзом системы: запрос на получение остатка денежных средств на счете, запрос и получение остатка денежных средств на пластиковой карте, запрос и получение комиссии по платежной операции и т.д.

  • "Тип ответа" - поиск по виду (наименованию) XML-сообщения, которое было получено системой ИКФЛ от внешней системы.

  • "Система" - система, которой адресовано сообщение (исходящее), или система, от которой получено сообщение (входящее).

  • "Период" - поиск по дате и времени, за которое были отправлены и получены сообщения системой;

  • "Слово в запросе" - поиск по слову в запросе,

  • "Слово в ответе" - поиск по слову в ответе,

  • "Приложение" - приложение, которое отправило сообщение во внешнюю систему.

В фильтре автоматически задается период выполненных действий за прошедшие сутки, но Вы можете задавать любой интересующий Вас период, за который были получены и отправлены сообщения.

Вы можете осуществлять поиск по подстроке в текстовых полях фильтра. Например, можно ввести в поля фильтра только несколько первых букв фамилии, имени или отчества пользователя.

Также Вы можете фильтровать список по любой комбинации полей. Например, можно отобрать сообщения по приложению, в котором совершена операция, и по слову в ответе.

После того как искомые параметры заданы, щелкните по кнопке Применить, и система покажет список интересующих Вас сообщений.

Если Вам необходимо выгрузить определенные записи журнала в файл, щелкните по кнопке Выгрузить, откроется окно, в котором нужно указать операцию Сохранить на диск и затем выбрать или ввести имя файла для сохранения.

Архивация журналов

С помощью пункта меню Архивация журналов можно вручную выгрузить из базы ИКФЛ протокол аудита в архив (протокол действий пользователей, протокол системных действий и протокол сообщений), можно загрузить в систему записи протокола из файла-архива. Архивация записей журналов используется для обеспечения быстродействия системы, а также для уменьшения размера базы данных.

В системе ИКФЛ также реализована возможность автоматической архивации протоколов, параметры настройки которой задаются в конфигурационном файле и в пункте меню Сервис.

Для того чтобы вручную выгрузить протокол из базы ИКФЛ, в поле "Вид журнала" выберите из списка журнал, который хотите заархивировать. Затем щелкните по кнопке В архив, и xml-файл будет сохранен в каталог, который указан в настройках системы. В результате записи за соответствующий период будут удалены из журнала и помещены в архив.

Примечание: периодичность архивации записей журнала системных действий задается в пункте меню Сервис – Настройка журнала системных действий. Аналогично периодичность архивации задается для журнала сообщений и журнала действий пользователей в соответствующих пунктах меню Сервис.

Для того чтобы загрузить журнал в базу ИКФЛ, в поле «Разархивация» щелкните по кнопке Обзор, выберите интересующий архив записей журнала и нажмите кнопку Из архива. В итоге протоколы загрузятся в базу, и с ними можно будет работать в штатном режиме (просматривать, фильтровать и т.д.).

Сменить пароль

Чтобы изменить пароль для входа в систему, в поле "Введите новый пароль" впишите Ваш новый пароль, потом подтвердите новый пароль, заполнив поле "Повторите новый пароль" той же комбинацией символов. Затем щелкните по кнопке Изменить, откроется окно, в котором будет написано "Пароль на вход в систему изменен". В результате при следующем входе в систему Вы должны будете указать в поле "Пароль" Ваш новый пароль.

Рабочие календари

С помощью пункта меню Рабочие календари задаются рабочие и нерабочие дни банка, т.е. для каждого вида платежа Вы можете указать дни приема документов и выходные дни, в которые проводка документов не осуществляется.

Выбор рабочего календаря зависит от внешней системы:

  • При работе с системой RS-Retail V6r20, используется календарь внешней системы. Чтобы настройка вступила в действие, установите флажок «Использовать календарь внешней системы». Данный календарь недоступен для изменения.

  • При работе с другими системами используются календари системы ИКФЛ, которые можно создавать, редактировать и удалять.

Создание календаря

Для того чтобы создать рабочий календарь, нажмите на кнопку Добавить, откроется страница редактирования календаря. На данной странице выполните следующие действия:

  • В поле «Название календаря» впишите наименование создаваемого рабочего календаря.

  • Чтобы отметить рабочий день как выходной, щелкните по цветовому обозначению выходного или праздничного дня в списке, после чего выделите в календаре интересующую Вас дату рабочего дня.

  • Чтобы указать выходной день как рабочий, щелкните по цветовому обозначению рабочего дня и отметьте в календаре интересующую Вас дату выходного дня.

  • Для перехода в календаре к предыдущему месяцу щелкните по кнопке <.

Шаблоны документов банка

Пункт меню Шаблоны документов банка предназначен для управления шаблонами документов банка.

Шаблон представляет собой документ формата Microsoft Word, Microsoft Excel или HTML, в котором есть статический текст и специальные теги. Статический текст остается неизменным, а теги при просмотре и печати документа заменяются информацией, которая берется из базы данных системы.

Для того чтобы создать шаблон документа, щелкните по кнопке Добавить шаблон, откроется страница создания шаблона документа. На данной странице в поле "Наименование шаблона" введите название шаблона, в поле "Описание шаблона" укажите сведения о шаблоне, в поле "Формат шаблона" выберите нужный формат, в поле "Имя файла" щелкните по кнопке Обзор, выберите файл с шаблоном, нажмите кнопку Сохранить, и шаблон документа появится в списке шаблонов документов. Если Вы не хотите создавать шаблон документа, щелкните по кнопке Список шаблонов, и система вернет Вас в список шаблонов документов.

Для изменения шаблона документа банка установите флажок напротив выбранного шаблона и нажмите на кнопку Редактировать или щелкните по наименованию шаблона, откроется страница для редактирования, на которой внесите необходимые изменения и нажмите на кнопку Сохранить.

Если Вы хотите удалить шаблон документа, установите галочку напротив интересующего Вас шаблона, щелкните по кнопке Удалить, и шаблон будет удален из списка.

Шаблоны документов банка можно группировать в пакеты шаблонов (группы шаблонов документов). См. раздел контекстной справки "Пакеты шаблонов документов банка".

Пакеты шаблонов документов банка

На данной странице в поле "Наименование" введите название пакета, в поле "Описание" укажите сведения о пакете, прикрепите один или несколько шаблонов документов, установив напротив них галочки, щелкните по кнопке Сохранить, и пакет шаблонов появится в списке. Если Вы не хотите создавать пакет шаблонов, щелкните по кнопке Список пакетов, и система вернет Вас в список пакетов шаблонов документов.

Вы можете отредактировать интересующие пакеты шаблонов документов банка. Для этого щелкните на названии пакета, откроется страница для редактирования, на которой внесите необходимые изменения и нажмите кнопку Сохранить.

Настройка журнала системных действий

Пункт меню Настройка журнала системных действий предназначен для управления уровнем детализации протокола. Уровень детализации - это объем информации, выводимый для каждой записи в журнале.

Вы можете задать свой уровень детализации для каждого модуля (обработчика расписаний, шлюза, ядра), по которым будет вестись протоколирование записей в журнале системных действий. В системе предусмотрены следующие уровни детализации:

  • "Максимальный" - в журнале отображаются записи обо всех действиях системы;

  • "Отладка" - в журнал попадают записи, содержащие оповещения о ходе работы различных лоадеров и синхронизаторов;

  • "Информация" - в журнале фиксируются записи, включающие информацию о запуске основных процессов и о ходе работы (обработанных документах, отправленных SMS и т.п.);

  • "Предупреждения" - в журнал попадают записи о предупреждениях системы;

  • "Ошибки" - в журнале фиксируются записи об ошибках;

  • "Выключен" - при выборе данного уровня журнал не ведется.

Чтобы задать уровень детализации, выберите из списка модуль и укажите интересующий Вас уровень детализации записей. Например, если Вы для модуля «Обработчик расписаний» выберете значение уровня детализации «Максимальный», то в журнале системных действий будут фиксироваться записи для данного модуля в полном объеме.

По умолчанию для каждого модуля задается значение уровня детализации «Ошибки».

После того как настройка уровня ведения записей задана, щелкните по кнопке Сохранить, и после перезапуска приложения настройка вступит в действие.

Также с помощью данного пункта меню Вы можете управлять параметрами автоматической и ручной архивации журнала системных действий.

Для перемещения в архив записей журнала необходимо указать следующие настройки:

  • Автоархивация

    Для того чтобы записи журнала системных действий архивировались автоматически, установите флажок в строке «Архивировать автоматически».

  • Периодичность автоархивации

    В поле «Периодичность» Вам нужно задать период (в днях или месяцах) и время автоматической архивации. Например, Вы указали «каждые 5 дней в 17:00», это означает, что система с периодичностью через каждые 5 дней в 17:00 выполняет выгрузку записей журнала системных действий в архив.

  • Период, записи за который остаются в журнале

    В поле «Оставлять в журнале записи за последние» Вам необходимо указать период (в днях), записи за который должны оставаться в журнале.

  • Путь к каталогу архива

    В поле «Каталог файлов архива» можно просмотреть путь к каталогу, в который система сохраняет архивы журнала. Путь к каталогу задается в настройках системы и недоступен для изменения через АРМ сотрудника банка.

Примечание: путь к каталогу для выгрузки файлов архива задается в конфигурационном файле. Описание данной настройки смотрите в пользовательской документации.

После того как параметры настройки архивации протоколов изменены, щелкните по кнопке Сохранить, и после сохранения настройка вступит в действие.

Настройка журнала действий пользователей

Пункт меню Настройка журнала действий пользователей позволяет для данного журнала задать степень и уровень детализации записей. Уровень детализации - это объем информации, выводимый для каждой записи в журнале.

Степень детализации

В системе можно выбрать следующие степени детализации записей журнала:

  • "Кратко" - запись журнала содержит только наименование действия пользователя;

  • "Средняя" - запись журнала состоит из наименования действия, выполненного пользователем, и URL операции;

  • "Подробно" - запись журнала включает в себя максимальную информацию о действии: наименование выполненного пользователем действия, URL операции и ее параметры.

Для того чтобы задать степень детализации, в поле «Степень детализации» выберите из выпадающего списка интересующее Вас значение. По умолчанию в системе задается значение «Кратко».

Уровень детализации

В системе можно задать следующие уровни детализации записей журнала:

  • "Выключен" - при выборе данного уровня журнал не ведется системой;

  • "Активные действия" - при выборе данного уровня в журнал попадают только записи об активных действиях пользователей, т.е. заведении клиентов, совершении платежей и др.;

  • "Все действия" - при выборе данного уровня в протокол попадают записи о совершении пользователем любых действий в системе.

Для того чтобы задать уровень детализации, в поле «Уровень детализации» выберите из выпадающего списка нужное значение. По умолчанию в системе задается значение «Активные действия».

Например, если Вы укажете степень детализации «Средняя» и уровень детализации «Все действия», то в протокол попадут записи о совершении пользователем всех действий, и запись будет состоять из наименования действия, выполненного пользователем, и URL операции.

После того как настройка уровня ведения записей задана, щелкните по кнопке Сохранить, и после перезапуска приложения настройка вступит в действие.

Также с помощью данного пункта меню Вы можете управлять параметрами автоматической и ручной архивации журнала действий пользователей.

Для перемещения в архив записей журнала необходимо указать следующие настройки:

  • Автоархивация

    Для того чтобы записи журнала действий пользователей архивировались автоматически, установите флажок в строке «Архивировать автоматически».

  • Периодичность автоархивации

    В поле «Периодичность» Вам нужно задать период (в днях или месяцах) и время автоматической архивации. Например, Вы указали «каждые 5 дней в 17:00», это означает, что система с периодичностью через каждые 5 дней в 17:00 выполняет выгрузку записей журнала действий пользователей в архив.

  • Период, записи за который остаются в журнале

    В поле «Оставлять в журнале записи за последние» Вам необходимо указать период (в днях), записи за который должны оставаться в журнале.

  • Путь к каталогу архива

    В поле «Каталог файлов архива» можно просмотреть путь к каталогу, в который система сохраняет архивы журнала. Путь к каталогу задается в настройках системы и недоступен для изменения через АРМ сотрудника банка.

Примечание: путь к каталогу для выгрузки файлов архива задается в конфигурационном файле. Описание данной настройки смотрите в пользовательской документации.

После того как параметры настройки архивации протоколов изменены, щелкните по кнопке Сохранить, и после сохранения настройка вступит в действие.
